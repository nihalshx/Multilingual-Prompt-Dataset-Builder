## 📌 Project Overview

**Multilingual Prompt Dataset Builder** is an end-to-end pipeline that creates, annotates, and publishes an open-source dataset of **700+ multilingual prompt-response pairs** across 7 languages and 5 task categories — all at zero cost using free-tier APIs.

### Why This Dataset Is Different

- ✅ **Human-verified Malayalam** — All Malayalam translations are reviewed and corrected by a native speaker, making this one of the very few open datasets with verified Malayalam prompt content.
- ✅ **Multi-dimensional quality scores** — Every prompt is scored on clarity, specificity, and completeness using a weighted custom schema.
- ✅ **Gemini-generated reference outputs** — Each prompt includes a high-quality reference answer generated by Gemini 1.5 Flash.
- ✅ **Ready for fine-tuning & evaluation** — Structured for direct use in LLM fine-tuning and benchmarking pipelines.

### At a Glance

|-------------------|-------------------------------------|
| Source Prompts    | 100 (hand-authored)                 |
| Languages         | 7 — EN, ML, HI, AR, ZH, ES, FR     |

| Quality Scoring   | 3-dimensional (clarity, specificity, completeness) |

## ⚙️ Pipeline Architecture

The pipeline executes in 5 sequential steps:

```
prompts.csv
     │
     ▼
[Step 1] translate.py        →  translated_prompts.csv  (EN + 6 languages)
     │
     ▼
[Step 2] generate_outputs.py →  dataset_with_outputs.csv (+ Gemini responses)
     │
     ▼
[Step 3] quality_score.py    →  dataset_scored.csv       (+ quality scores)
     │
     ▼
[Step 4] quality_score.py    →  malayalam_review.csv     (human review export)
     │      (manual review)
     ▼
[Step 5] quality_score.py    →  dataset_final.csv        (merged final dataset)
     │
     ▼
[Step 6] upload_hf.py        →  Hugging Face Hub 🚀
```

## 🚀 Quick Start

### 1. Clone the Repository

```bash
git clone https://github.com/nihalshx/Multilingual-Prompt-Dataset-Builder.git
cd Multilingual-Prompt-Dataset-Builder
```

### 2. Create a Virtual Environment

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

### 3. Configure API Keys

API keys are entered through the **Streamlit Settings tab** at runtime — no `.env` file needed.

| Key                    | Where to Get It                                          | Required? |
|------------------------|----------------------------------------------------------|-----------|
| `GEMINI_API_KEY`       | https://makersuite.google.com/app/apikey                 | ✅ Yes    |
| `HF_TOKEN`             | https://huggingface.co/settings/tokens (Write token)    | ✅ Yes    |
| `GOOGLE_CLOUD_API_KEY` | https://console.cloud.google.com → Cloud Translation API | ⚡ Optional (improves translation quality) |

## 🏃 Running the Pipeline

### Option A — Run Everything at Once (Recommended)

```bash
python run_pipeline.py
```

The script runs Steps 1–4 automatically, then **pauses** for the native-speaker Malayalam review. After you complete the review CSV, run:

```bash
python run_pipeline.py --step merge
```

### Option B — Run Individual Steps

```bash
python run_pipeline.py --step translate   # Step 1: Translate into 6 languages
python run_pipeline.py --step outputs     # Step 2: Generate Gemini outputs
python run_pipeline.py --step score       # Step 3: Compute quality scores
python run_pipeline.py --step review      # Step 4: Export Malayalam review CSV

# → Complete the review in data/malayalam_review.csv

python run_pipeline.py --step merge       # Step 5: Merge review back in
```

### Step 4 — Malayalam Native-Speaker Review

After running `--step review`, open `data/malayalam_review.csv` in Excel or Google Sheets. For each row:

- If the auto-translation is accurate → set `ml_review_status = approved`
- If it needs correction → set `ml_review_status = corrected`, paste the corrected Malayalam text into `prompt_ml_corrected`

> 💡 This human-verification step is the biggest differentiator of this dataset.

## 📊 Quality Scoring Schema

Each prompt is scored on a 0–1 scale using a weighted formula:

| Dimension      | Weight | Description                                              |
|----------------|--------|----------------------------------------------------------|
| Clarity        | 40%    | How unambiguous and well-structured the prompt is        |
| Specificity    | 35%    | How precisely the prompt defines the task                |
| Completeness   | 25%    | Whether the prompt provides all required context         |

**Task-specific word count norms** are applied when scoring Gemini outputs:

| Task Category        | Min Words | Ideal Words | Max Words |
|----------------------|-----------|-------------|-----------|
| Summarisation        | 30        | 80          | 250       |
| Question Answering   | 20        | 60          | 200       |
| Creative Writing     | 40        | 120         | 400       |
| Instruction Following| 10        | 30          | 120       |
| Reasoning            | 20        | 50          | 150       |

## 🐛 Troubleshooting

**`googletrans` returns empty strings?**
The free library hits rate limits. Increase `TRANSLATE_DELAY` in `config.py` to `1.5`, or add `GOOGLE_CLOUD_API_KEY` to your environment to use the Cloud API.

**Gemini rate limit errors?**
The free tier allows ~15 requests/minute. Increase `GEMINI_REQUEST_DELAY` in `config.py` to `5.0`. All pipeline steps checkpoint automatically — safe to interrupt and resume.

**How do I add more prompts?**
Add rows to `data/prompts.csv` maintaining the existing column structure. No code changes are needed.

**Can I add more languages?**
Add entries to the `LANGUAGES` dict in `config.py`. The pipeline, scoring, and upload steps all update automatically.

**The Streamlit app won't start?**
Ensure you've run at least Step 1 of the pipeline (`--step translate`) so the data files exist.
